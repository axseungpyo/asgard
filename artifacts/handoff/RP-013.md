## Known Issues
- macOS 기본 환경에 `timeout`/`gtimeout`이 없어 Loki 스크립트 smoke test는 watchdog-only 보호로 실행됐다. 이는 `shared/context.md`의 기존 제약과 동일하다.
- `npx tsc --noEmit`와 `npm run build`를 병렬 실행하면 `.next/types` 산출물 경합으로 일시 실패할 수 있다. 순차 실행 시 정상 통과한다.
- Loki는 아직 mock 에이전트라 실제 이미지 파일이나 URL을 생성하지 않고 placeholder acknowledgement만 남긴다.

## Recommended Next Actions
1. 실제 이미지 생성 백엔드가 정해지면 `scripts/delegate-loki.sh`의 placeholder 실행부를 API 호출로 교체한다.
2. Loki 작업 결과를 대시보드에서 미리보기할 필요가 생기면 별도 UI TP로 image artifact viewer를 추가한다.
3. 필요하면 `tsconfig`와 build/typecheck 워크플로를 분리해 `.next/types` 경합을 제거한다.

## Context Digest
- Architecture: Yggdrasil은 이제 `odin`, `brokkr`, `heimdall`, `loki` 4개 에이전트를 단일 상수/타입 소스로 관리한다.
- Key entry points: Loki 등록은 `src/yggdrasil/dashboard/lib/constants.ts`, 상태 감시는 `src/yggdrasil/server/agents.ts`/`src/yggdrasil/server/watcher.ts`, 위임은 `scripts/delegate-loki.sh`, Odin 호출은 `.claude/skills/delegate-loki/SKILL.md`.
- Config/env: 새 npm 패키지는 추가하지 않았다. Loki mock 실행에는 추가 자격 증명이 필요 없다.
- Gotchas: Loki TP smoke test는 동일한 `RP-013.md`를 먼저 mock 내용으로 생성하므로, 실제 Saga 제출 시 최종 RP로 다시 써야 한다.
