# RP-022: AI Brain — LLM Gateway + Smart Odin

## Summary
LLM 포트(`ILLMGateway`)와 Claude/Regex gateway를 추가하고, Odin 명령 처리 경로를 AI 우선 후 regex 폴백 구조로 확장했다.
`ProcessCommandUseCase`는 Claude 가용성 체크, 프로젝트 컨텍스트 빌드, Claude tool_use 실행 연결, API 오류 시 regex graceful fallback을 수행한다.

## Known Issues / Follow-ups
- 실제 `npm install`은 네트워크 차단으로 실패했다. 현재 `node_modules/@anthropic-ai/sdk` 임시 shim으로 타입체크/부팅 안정성만 보장한다.
- 샌드박스가 포트 바인딩을 막아 `/api/health` 실제 curl 검증은 수행하지 못했다.
- `dashboard`의 API key 입력은 아직 브라우저 `localStorage`에만 저장되므로, 서버가 읽는 `artifacts/config/mcp-servers.json`과의 완전한 연동은 후속 서버 설정 저장 작업이 필요하다.

## Recommended Next Actions
1. 네트워크 가능한 환경에서 `npm install`을 다시 실행해 실제 `@anthropic-ai/sdk`를 설치한다.
2. 포트 바인딩 가능한 환경에서 `curl localhost:7777/api/health`를 재검증한다.
3. 대시보드 Settings의 Anthropic key를 서버 설정 파일로 저장하는 경로를 별도 TP로 연결한다.

## Context Digest (for Odin)
- Architecture: Odin 명령 처리가 `ClaudeLLMGateway -> tool_use -> skill execution` 또는 `RegexFallbackGateway -> execute_skill` 구조로 확장됨.
- Key entry points: `server/core/use-cases/odin/ProcessCommandUseCase.ts`, `server/adapters/gateways/ClaudeLLMGateway.ts`, `server/core/use-cases/odin/ContextBuilder.ts`.
- Config/env: `ANTHROPIC_API_KEY` 우선, 없으면 `artifacts/config/mcp-servers.json` 또는 `.claude/settings.json`에서 탐색.
- Gotchas: 실제 npm registry 접근이 막힌 환경에서는 임시 shim이 동작하며, Claude 경로는 런타임 에러 시 regex fallback으로 내려간다.
