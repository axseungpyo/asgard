# TP-013: Loki 에이전트 기반 구조 + 대시보드 통합

## Metadata
- Agent Target: codex
- Complexity: medium
- Priority: P3
- Created: 2026-03-09

## Objective

Loki(이미지 생성 에이전트)의 기반 인프라를 구축한다. 실제 이미지 생성 API 연동은 하지 않고, 에이전트 등록/모니터링/위임 인프라만 구현한다.

## Scope In

### 1. 에이전트 등록 (`dashboard/lib/constants.ts`)
- AGENT_NAMES에 `"loki"` 추가
- AGENT_CONFIG에 Loki 설정 추가:
  - displayName: `"Loki"`
  - model: `"Image Gen"`
  - color: `"#a855f7"` (보라색)

### 2. 에이전트 타입 확장 (`dashboard/lib/types.ts`)
- AgentName 타입에 `"loki"` 추가

### 3. delegate-loki.sh 스크립트 (`scripts/delegate-loki.sh`)
- Brokkr/Heimdall 위임 스크립트 구조 참조
- 기본 구조:
  1. TP 파일 읽기
  2. 이미지 생성 작업을 위한 placeholder (실제 API 미연동)
  3. RP 파일 생성 (이미지 URL/경로 포함)
  4. Watchdog 보호 (300s 타임아웃)
  5. mkdir 기반 동시 실행 Lock
  6. 로그 로테이션
- 현재는 모의 실행: "Loki image generation is not yet connected to an API. TP-NNN acknowledged." 출력 후 RP 생성

### 4. `/delegate-loki` Skill 정의
- `.claude/skills/delegate-loki/SKILL.md` 생성
- Odin이 `/delegate-loki TP-NNN`으로 호출
- 내부적으로 `bash scripts/delegate-loki.sh TP-NNN` 실행

### 5. 대시보드 통합
- AgentCard가 4번째 에이전트(Loki)를 렌더링
- TerminalPanel이 Loki 로그를 분리 표시
- FlowView/StatsPanel에서 Loki 에이전트 데이터 표시

### 6. CLAUDE.md/AGENTS.md 업데이트
- Agent Routing Table에 Loki 추가
- Mode 시스템에 Loki 모드 추가 (Sketch/Canvas)

## Scope Out
- 실제 이미지 생성 API (Nano Banana, DALL-E 등) 연동은 하지 않는다
- 이미지 미리보기 UI는 하지 않는다
- 새 npm 패키지를 설치하지 않는다

## Relevant Files
- `src/yggdrasil/dashboard/lib/constants.ts`
- `src/yggdrasil/dashboard/lib/types.ts`
- `src/yggdrasil/server/agents.ts`
- `scripts/delegate-codex.sh` — 참조 구조
- `CLAUDE.md`, `AGENTS.md`

## Acceptance Criteria
1. `constants.ts`에 Loki가 AGENT_NAMES와 AGENT_CONFIG에 등록되어 있다
2. `types.ts`의 AgentName에 `"loki"`가 포함된다
3. `scripts/delegate-loki.sh`가 존재하고 실행 가능하다 (chmod +x)
4. `.claude/skills/delegate-loki/SKILL.md`가 존재한다
5. 대시보드 Overview에서 4개 에이전트 카드가 표시된다 (Odin, Brokkr, Heimdall, Loki)
6. 기존 테스트 전부 PASS한다
7. TypeScript 컴파일 에러가 없다

## Implementation Notes
- Loki 모드: Sketch [Fast] — 빠른 스케치, Canvas [Standard] — 정밀 이미지 (현재 모의만)
- delegate-loki.sh는 delegate-codex.sh의 Lock/로그 로테이션/Watchdog 로직을 그대로 복사하되, 실행 부분만 placeholder로 처리
- constants.ts의 AGENT_NAMES 배열 순서: odin, brokkr, heimdall, loki
- agents.ts의 getAgentStates()가 loki PID 파일도 감시하도록 확인 (이미 동적이면 추가 작업 불필요)
- Pino logger 관련 변경 불필요 (이미 동적 에이전트명 처리)
