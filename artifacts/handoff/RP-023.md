# RP-023: Event Bus + Domain Events

## Summary
`IEventBus` 포트와 `core/events/` 도메인 이벤트 타입을 추가하고, `InProcessEventBus`를 Node.js `EventEmitter` 기반으로 구현했다.  
Watcher는 파일/로그 변화를 EventBus로 발행하고, WebSocket 핸들러는 EventBus를 구독해 기존 실시간 메시지 포맷을 유지하도록 전환했다.  
Use Case는 성공 시 도메인 이벤트를 발행하도록 확장했고, subscriber 에러가 다른 subscriber에 전파되지 않도록 격리했다.

## Known Issues / Follow-ups
- 샌드박스 환경에서 포트 바인드가 금지되어 `/api/health`는 코드와 무관하게 검증 불가했다.
- `status-handler`는 use case 이벤트와 watcher file 이벤트를 함께 받아 실시간 갱신을 유지하며, 중복 전송은 25ms coalescing으로 완화했다.
- `ProcessApprovalUseCase`의 결과는 TP-023 scope대로 직접 broadcast 경로를 유지했다. 필요하면 후속 TP에서 approval 이벤트도 도메인 이벤트로 승격 가능하다.
