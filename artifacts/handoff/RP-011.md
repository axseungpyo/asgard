# RP-011: 에이전트 메트릭 수집 및 시각화

## Summary
`execution.log`와 `artifacts/INDEX.md`를 직접 파싱하는 메트릭 수집 모듈을 추가하고, `/api/metrics` 엔드포인트로 집계 결과를 노출했다.
Stats 탭에 `MetricsPanel`을 연결해 에이전트 성공률 바 차트, 일별 완료 차트, 최근 실행 이력 테이블, 평균 실행 시간을 렌더링하도록 구현했다.
단위 테스트 4건을 추가했고 기존 테스트 포함 전체 Vitest와 TypeScript 컴파일 검증을 통과했다.

## Known Issues
- `execution.log`에는 현재 `[brokkr] START TP-011` 형식과 예시의 `[codex] TP-008 started` 형식이 혼재해 있어 두 포맷을 모두 지원했다. 이외의 자유 형식 shell 로그 라인은 메트릭에서 무시한다.
- 메트릭은 요청 시점 파일 파싱 기반이다. 실시간 스트리밍은 TP Scope Out에 따라 추가하지 않았다.

## Recommended Next Actions
1. 대시보드 WebSocket 이벤트에 메트릭 재요청을 연결해 Stats 탭 새로고침 없이 최신 집계를 반영할지 검토한다.
2. 향후 `BLOCKED` 로그 포맷이 실제로 기록되면 회귀 테스트에 샘플을 추가해 blocked 실행 이력 노출을 고정한다.

## Context Digest
- Architecture: 별도 DB 없이 `artifacts/INDEX.md`와 `artifacts/logs/execution.log`를 파싱하는 서버 집계 계층을 추가했다.
- Key entry points: `/api/metrics`, `collectMetrics()`, `MetricsPanel`, `StatsPanel`.
- Config/env: 신규 환경 변수나 패키지 추가 없음.
- Gotchas: 에이전트명은 `codex -> brokkr`, `gemini -> heimdall`로 정규화해 INDEX와 실행 로그 데이터를 합산한다.
