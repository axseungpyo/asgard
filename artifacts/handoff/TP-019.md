## Objective

## Scope In

### Part 1: WebSocket 분리

#### 신규 파일
- `server/websocket/ws-manager.ts` — WSS 생성, 하트비트 관리, 인증(authorizeWebSocket), broadcast 함수
- `server/websocket/logs-handler.ts` — wssLogs connection 핸들러
- `server/websocket/status-handler.ts` — wssStatus connection 핸들러
- `server/websocket/odin-handler.ts` — wssOdin connection 핸들러
- `server/websocket/index.ts` — WebSocket 초기화 함수 export (setupWebSockets)

#### index.ts 변경
- WebSocket 관련 코드 전체를 제거하고, `setupWebSockets(server, watcher, asgardRoot)` 호출로 대체
- index.ts는 Express + Next.js 초기화 + 서버 리스닝만 담당

## Implementation Notes
1. ws-manager.ts의 setupWebSockets 시그니처: `setupWebSockets(server: http.Server, watcher: AsgardWatcher, asgardRoot: string): void`
2. 각 handler 파일은 connection 핸들러 함수만 export: `export function handleLogsConnection(ws, request, watcher, authorizeWs, broadcast)`
3. 컴포넌트 이동 시 각 그룹에 `index.ts` barrel export는 만들지 않음 — 직접 import 경로 사용 (barrel은 Next.js 트리쉐이킹에 불리).
4. 기존 컴포넌트 내부의 상대 import (예: `./lib/auth`, `./lib/types`)는 `../lib/auth`, `../lib/types`로 변경됨.
