## Objective

TP-024에서 추가한 파일/CLI Tool에 **안전장치**를 강화한다:
1. **write_file 승인 플로우** — 파일 쓰기 전 사용자 승인 요청
2. **경로 화이트리스트** — 설정 기반 접근 허용/차단 경로 관리
3. **Tool 실행 로깅** — 모든 Tool 호출을 EventBus 이벤트로 기록
4. **read_file 크기 제한** — 대용량 파일 방어 개선

---

## Context

### 현재 상태 (TP-024 완료 후)

- `FileSystemToolExecutor`에 기본 경로 검증 존재 (프로젝트 밖, .env, node_modules, .git 차단)
- `write_file`은 승인 없이 바로 실행됨 — **위험**
- Tool 실행 이벤트가 EventBus에 발행되지 않음 — 감시 불가
- 경로 차단 리스트가 코드에 하드코딩됨 — 설정으로 외부화 필요

### 목표

```
Tool 호출
  ↓
[1] 경로 검증 (화이트리스트/블랙리스트)
  ↓
[2] 위험도 판별 (read=safe, write=dangerous)
  ↓
[3] dangerous → 사용자 승인 요청 (approval_request)
  ↓
[4] 승인 완료 → 실행
  ↓
[5] EventBus에 TOOL_EXECUTED 이벤트 발행
```

### 3. adapters/tools/FileSystemToolExecutor.ts 수정 (~40줄 추가)

변경 사항:
- **write_file**: `execute()` 시 바로 쓰지 않고 `{ requiresApproval: true, approvalDescription: "파일 쓰기: {path}" }` 반환
- **경로 블랙리스트 외부화**: 생성자에서 `blockedPaths: string[]` 받기
- **read_file 크기 선검사**: `fs.stat()`으로 파일 크기 확인, 1MB 이상 시 에러
- **EventBus 연동**: 생성자에서 `IEventBus` 받아 실행 후 이벤트 발행

```typescript
export class FileSystemToolExecutor implements IToolExecutor {
  constructor(
    private readonly fileSystem: IFileSystem,
    private readonly projectRoot: string,
    private readonly eventBus?: IEventBus,       // 추가
    private readonly blockedPaths?: string[],     // 추가 (기본: [".env", "node_modules", ".git"])
  ) {}

  async execute(toolCall: LLMToolCall, projectRoot: string): Promise<ToolResult> {
    const startTime = Date.now();
    try {
      const result = await this.executeInternal(toolCall, projectRoot);
      this.publishEvent(toolCall, result, Date.now() - startTime);
      return result;
    } catch (err) {
      const errorResult = { success: false, output: "", error: err.message };
      this.publishEvent(toolCall, errorResult, Date.now() - startTime);
      return errorResult;
    }
  }

  private async executeInternal(toolCall: LLMToolCall, projectRoot: string): Promise<ToolResult> {
    // write_file → requiresApproval: true
    if (toolCall.name === "write_file") {
      const resolvedPath = this.validatePath(...);
      return {
        success: true,
        output: "",
        requiresApproval: true,
        approvalDescription: `파일 쓰기 승인 필요: ${path.relative(projectRoot, resolvedPath)}`,
      };
    }
    // ... 나머지 tool은 기존 로직
  }
}
```

  // 1. 기존 approval 도구 (delegate, stop) 처리

    // NEW: executor가 승인 요청을 반환한 경우
    if (result.requiresApproval) {
      const approvalId = `tool-${Date.now()}-${Math.random().toString(36).slice(2, 8)}`;
      this.approvalStore.set(approvalId, {
        skill: `tool:${toolCall.name}`,
        args: JSON.stringify(toolCall.input),
      });
      messages.push(this.messageRepository.addMessage({
        role: "odin",
        type: "approval_request",
        content: result.approvalDescription || "Tool 실행 승인이 필요합니다.",
        actions: [
          { id: approvalId, label: "Approve", type: "approve" },
          { id: `${approvalId}-reject`, label: "Cancel", type: "reject" },
        ],
        metadata: { tool: toolCall.name },
      }));
      return;
    }

    // 성공/실패 메시지
    messages.push(...);
    return;
  }

## Acceptance Criteria

1. **AC-1**: `write_file` Tool 호출 시 `requiresApproval: true` 반환, 실제 파일 쓰기 안 함
2. **AC-2**: 사용자 승인 후 `executeApproved()`로 실제 파일 쓰기 실행
3. **AC-3**: `read_file` — 1MB 이상 파일 접근 시 에러 메시지 반환
4. **AC-4**: 모든 Tool 실행 시 `TOOL_EXECUTED` 이벤트가 EventBus에 발행됨
5. **AC-5**: 이벤트에 tool명, input, success, durationMs, timestamp 포함
6. **AC-6**: 기존 경로 안전장치(프로젝트 밖, .env, node_modules, .git 차단) 유지
