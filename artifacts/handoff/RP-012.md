# RP-012: WebSocket 인증 + 토큰 기반 API 보호

## Summary
Yggdrasil 서버에 Bearer 토큰 기반 인증을 추가했다. 서버는 `artifacts/.yggdrasil-token`을 생성·재사용하고, `/api/health`를 제외한 `/api/*`와 `/ws/*` 연결을 토큰으로 보호한다.

대시보드는 로컬 스토리지 토큰을 사용해 인증된 `fetch`와 WebSocket 연결을 수행하도록 변경했고, 토큰이 없거나 유효하지 않으면 입력 모달을 먼저 표시한다.

## Context Digest
- Architecture: 서버 공통 인증 로직을 `server/auth.ts`로 분리하고 HTTP/WS 양쪽에서 재사용한다.
- Key entry points: `src/yggdrasil/server/index.ts`, `src/yggdrasil/server/routes.ts`, `src/yggdrasil/dashboard/app/page.tsx`
- Config/env: `YGGDRASIL_AUTH=false`면 인증을 우회한다. 토큰 저장 키는 `yggdrasil-auth-token`이다.
- Gotchas: `/api/health`만 무인증이며, 나머지 대시보드 API와 `/ws/logs`, `/ws/status`는 토큰이 필요하다.
