# TP-012: WebSocket 인증 + 토큰 기반 API 보호

## Metadata
- Agent Target: codex
- Complexity: medium
- Priority: P3
- Created: 2026-03-09

## Objective

Yggdrasil 서버에 토큰 기반 인증을 추가하여, WebSocket과 API 엔드포인트를 무단 접근으로부터 보호한다.

## Scope In

### 1. 토큰 관리 (`server/auth.ts`)
- 서버 최초 실행 시 `artifacts/.yggdrasil-token` 파일에 랜덤 토큰 생성 (존재하면 재사용)
- `crypto.randomBytes(32).toString('hex')` 사용
- `validateToken(token: string): boolean` 함수 export
- `getToken(): string` 함수 export (서버 시작 시 콘솔에 토큰 출력)
- Pino logger 사용

### 2. API 인증 미들웨어
- `authMiddleware` Express 미들웨어 생성
- `Authorization: Bearer <token>` 헤더 검증
- 인증 실패 시 `401 Unauthorized` 반환
- 적용 대상: `/api/*` (단, `/api/health`는 인증 없이 접근 가능)
- 환경변수 `YGGDRASIL_AUTH=false`로 인증 비활성화 가능 (개발 편의)

### 3. WebSocket 인증
- WS 연결 시 URL 쿼리 파라미터로 토큰 전달: `ws://host:7777/ws/logs?token=<token>`
- `wss.on('connection')` 핸들러에서 토큰 검증
- 인증 실패 시 `ws.close(4001, 'Unauthorized')` 호출
- 기존 WebSocket 로직은 인증 성공 후에만 실행

### 4. 대시보드 토큰 입력
- 대시보드 최초 로드 시 `localStorage`에 저장된 토큰 확인
- 토큰 없거나 유효하지 않으면 간단한 토큰 입력 UI 표시
- `fetch` 호출 시 `Authorization: Bearer <token>` 헤더 추가
- WebSocket URL에 `?token=<token>` 쿼리 추가
- 토큰 입력 UI: 단순 모달 (토큰 입력 필드 + 연결 버튼)

### 5. 단위 테스트
- `server/__tests__/auth.test.ts` 생성
- 테스트 케이스:
  - 올바른 토큰으로 인증 성공
  - 잘못된 토큰으로 인증 실패
  - 토큰 없이 인증 실패
  - YGGDRASIL_AUTH=false 시 인증 우회

## Scope Out
- HTTPS/TLS 설정은 하지 않는다 (리버스 프록시에서 처리)
- OAuth/JWT 등 복잡한 인증은 하지 않는다 — 단순 Bearer 토큰
- 사용자 관리 (다중 사용자)는 하지 않는다
- 새 npm 패키지를 설치하지 않는다

## Relevant Files
- `src/yggdrasil/server/routes.ts` — 미들웨어 적용
- `src/yggdrasil/server/index.ts` — WS 인증, 서버 시작 시 토큰 출력
- `src/yggdrasil/server/logger.ts` — Pino logger
- `src/yggdrasil/dashboard/lib/websocket.ts` — WS URL에 토큰 추가
- `src/yggdrasil/dashboard/app/page.tsx` — 토큰 입력 UI, fetch 헤더

## Acceptance Criteria
1. `server/auth.ts`가 존재하고 `validateToken`, `getToken` 함수를 export한다
2. 토큰 없이 `GET /api/status` 호출 시 `401` 응답을 반환한다
3. `GET /api/health`는 토큰 없이도 `200` 응답을 반환한다
4. `YGGDRASIL_AUTH=false` 환경변수 설정 시 인증이 비활성화된다
5. `server/__tests__/auth.test.ts`에 최소 4개 테스트가 있고 전부 PASS한다
6. 기존 테스트 포함 전체 테스트가 PASS한다
7. TypeScript 컴파일 에러가 없다

## Implementation Notes
- 토큰 파일 경로: `path.join(asgardRoot, 'artifacts', '.yggdrasil-token')`
- `.gitignore`에 `.yggdrasil-token` 추가 필요
- 대시보드에서 fetch wrapper 함수 생성: `authFetch(url, options)` — 토큰 자동 첨부
- localStorage 키: `yggdrasil-auth-token`
- Pino logger 사용 (server/logger.ts의 createLogger)
