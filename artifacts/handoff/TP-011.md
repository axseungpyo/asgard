# TP-011: 에이전트 메트릭 수집 및 시각화

## Metadata
- Agent Target: codex
- Complexity: complex
- Priority: P3
- Created: 2026-03-09

## Objective

에이전트별 실행 시간, 성공률, TP 처리량 등 핵심 메트릭을 수집하고 대시보드에서 시각화한다.

## Scope In

### 1. 메트릭 수집 모듈 (`server/metrics.ts`)
- 기존 `execution.log`와 `INDEX.md` 파싱으로 메트릭 추출 (별도 DB 사용 안 함)
- 수집 메트릭:
  - **에이전트별 총 TP 수**: completed, blocked 구분
  - **에이전트별 성공률**: completed / (completed + blocked) * 100
  - **에이전트별 평균 실행 시간**: execution.log의 시작/종료 타임스탬프 차이
  - **일별 TP 완료 수**: INDEX.md Completed 테이블의 날짜 기반
  - **최근 실행 이력**: 마지막 10건의 실행 기록 (에이전트, TP, 시간, 결과)
- `collectMetrics(): Promise<AgentMetrics>` 함수 export
- Pino logger 사용

### 2. API 엔드포인트
- `GET /api/metrics` — 전체 메트릭 JSON 반환
  - 응답 타입:
    ```
    {
      agents: { name: string, totalTasks: number, completed: number, blocked: number, successRate: number, avgDuration: number }[],
      daily: { date: string, count: number }[],
      recentExecutions: { agent: string, tp: string, duration: number, result: string, timestamp: string }[]
    }
    ```

### 3. 대시보드 컴포넌트 (`dashboard/components/MetricsPanel.tsx`)
- Stats 탭 확장 또는 별도 섹션으로 배치
- 시각화 요소:
  - 에이전트별 성공률 바 차트 (CSS 기반)
  - 일별 TP 완료 수 미니 차트 (CSS 기반 spark line 또는 bar)
  - 최근 실행 이력 테이블 (에이전트, TP, 소요시간, 결과)
  - 에이전트별 평균 실행 시간 표시
- 외부 차트 라이브러리 사용 금지 (CSS/SVG만)

### 4. 단위 테스트
- `server/__tests__/metrics.test.ts` 생성
- 테스트 케이스:
  - execution.log 파싱 (정상 로그)
  - 빈 로그 처리
  - 일별 집계 계산
  - 성공률 계산

## Scope Out
- 실시간 메트릭 스트리밍 (WebSocket)은 하지 않는다 — 요청/응답 방식
- 외부 모니터링 시스템 (Prometheus, Grafana) 연동은 하지 않는다
- 새 npm 패키지를 설치하지 않는다

- `src/yggdrasil/server/logger.ts` — Pino logger

## Acceptance Criteria
1. `server/metrics.ts`가 존재하고 `collectMetrics` 함수를 export한다
2. `GET /api/metrics`가 agents, daily, recentExecutions 구조의 JSON을 반환한다
3. `MetricsPanel.tsx`가 존재하고 Stats 탭에서 렌더링된다
4. 에이전트별 성공률 바 차트가 CSS로 구현되어 있다
5. `server/__tests__/metrics.test.ts`에 최소 4개 테스트가 있고 전부 PASS한다
6. 기존 테스트 포함 전체 테스트가 PASS한다
7. TypeScript 컴파일 에러가 없다

## Implementation Notes
- execution.log 형식 예시: `2026-03-09 21:30 [codex] TP-008 started` / `2026-03-09 21:37 [codex] TP-008 done (431s)`
- 로그 파싱 시 정규식으로 에이전트명, TP-ID, 결과, 시간 추출
- 빈 execution.log 또는 파일 없음 시 빈 메트릭 반환 (에러 아님)
- StatsPanel에 메트릭 데이터를 props로 전달하거나, MetricsPanel을 별도 컴포넌트로 Stats 탭 하단에 배치
- Pino logger 사용 (server/logger.ts의 createLogger)
