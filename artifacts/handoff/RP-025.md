# RP-025: Tool Safety — 승인 플로우 + 경로 화이트리스트 + 실행 로깅

## Summary
TP-025 범위 내에서 파일 시스템 Tool 안전장치를 강화했다. `write_file`은 승인 요청으로 전환했고, 승인 후 `executeApproved()`로만 실제 쓰기가 수행되도록 연결했으며, 모든 Tool 실행은 `tool.executed` 이벤트로 기록되도록 했다.

또한 `read_file`에 1MB 사전 크기 제한을 추가했고, 관련 회귀/신규 테스트를 보강했다.

## Known Issues / Follow-ups
- 현재 실행 환경에서 포트 바인딩이 차단되어 `/api/health` 런타임 검증은 수행하지 못했다.
- TP 본문에 언급된 경로 화이트리스트 전체 외부 설정화는 현재 코드 구조상 `blockedPaths` 생성자 주입까지 반영했다. 별도 설정 파일 연동은 TP의 필수 수정 지시에는 포함되지 않아 확장하지 않았다.

## Recommended Next Actions
1. 포트 바인딩 가능한 환경에서 `node --import tsx server/index.ts` 후 `/api/health`를 재검증한다.
2. 필요 시 `blockedPaths`를 환경설정 계층으로 끌어올려 운영 설정 가능하게 확장한다.

## Context Digest (for Odin)
- Architecture: Tool 실행 계약에 승인 단계와 실행 이벤트가 추가되었다.
- Key entry points: `ProcessCommandUseCase.handleToolCall()`, `ProcessApprovalUseCase.execute()`, `FileSystemToolExecutor.execute()/executeApproved()`
- Config/env: `FileSystemToolExecutor`는 선택적 `blockedPaths`와 `eventBus`를 받는다.
- Gotchas: `write_file`은 이제 `execute()`만으로는 실제 파일을 쓰지 않으며, 승인 후 경로로만 반영된다.
