## Objective

## Scope In

### 1. core/ports/ILLMGateway.ts — LLM 추상화 인터페이스
```typescript
interface LLMMessage {
  role: "user" | "assistant" | "system";
  content: string;
}

interface LLMToolCall {
  name: string;
  input: Record<string, unknown>;
}

interface LLMResponse {
  content: string;
  toolCalls?: LLMToolCall[];
  stopReason: "end_turn" | "tool_use" | "max_tokens";
}

interface ILLMGateway {
  isAvailable(): boolean;
  chat(messages: LLMMessage[], options?: { tools?: LLMToolDefinition[]; system?: string }): Promise<LLMResponse>;
}
```

### 2. adapters/gateways/ClaudeLLMGateway.ts — Claude API 구현체
- `@anthropic-ai/sdk` 패키지 사용 (package.json에 추가)
- API Key는 환경변수 `ANTHROPIC_API_KEY` 또는 Settings(`artifacts/config/mcp-servers.json`)에서 읽기
- 기본 모델: `claude-sonnet-4-6` (비용 효율적)
- isAvailable(): API 키가 설정되었는지 확인
- chat(): Anthropic SDK의 messages.create() 호출

### 3. adapters/gateways/RegexFallbackGateway.ts — 폴백 구현체
- ILLMGateway를 implements하되, 기존 regex 스킬 매칭 로직을 사용
- isAvailable(): 항상 true
- chat(): 입력 메시지에서 regex로 의도 파악 후 정형화된 응답 반환

### 4. core/use-cases/odin/ProcessCommandUseCase.ts 확장
- ILLMGateway 의존성 추가
- 실행 흐름:
  1. LLM Gateway가 available이면 AI 모드로 처리
  2. AI 응답에서 tool_use가 있으면 해당 스킬 실행
  3. AI가 unavailable이면 기존 ISkillRegistry(regex) 폴백
- AI 시스템 프롬프트에 프로젝트 컨텍스트 포함:
  - 현재 에이전트 상태
  - 활성 태스크 목록
  - 사용 가능한 스킬 목록

### 5. Context Builder

## Implementation Notes

### 의존성 흐름
```
ProcessCommandUseCase
  ├→ ILLMGateway (port)
  │   ├→ ClaudeLLMGateway (adapter) — API 키 있을 때
  │   └→ RegexFallbackGateway (adapter) — API 키 없을 때
  ├→ IMessageRepository (port)
  ├→ ISkillRegistry (port)
  ├→ ITaskRepository (port) — 컨텍스트 빌딩용
  └→ IAgentRepository (port) — 컨텍스트 빌딩용
```

### AI 시스템 프롬프트 구조
```
You are Odin, the Brain Agent of the Asgard orchestration system.
You plan tasks, delegate to agents (Brokkr/Heimdall), and review results.

Current Project State:
- Active Tasks: [list from ITaskRepository]
- Agent Status: [list from IAgentRepository]
- Available Skills: [list from ISkillRegistry]

Rules:
1. Use tools to take actions (never just describe what you would do)
2. For dangerous actions (delegate, stop), use ask_user tool first
3. Keep responses concise and in Korean
```

### 안전장치
- AI 응답의 tool_use에서 delegate/stop은 항상 ask_user를 먼저 호출하도록 프롬프트에 명시
- 1회 요청당 최대 tool_use 3회 제한 (무한 루프 방지)
- API 에러 발생 시 regex 폴백으로 전환 (graceful degradation)
