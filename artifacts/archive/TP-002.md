# TP-002: Yggdrasil Phase 1 — Backend Server + 기본 Dashboard UI

## Agent Target
codex

## Complexity Hint
extreme

## Objective
Asgard 에이전트의 실시간 상태/로그를 모니터링하는 Yggdrasil 웹 대시보드의 백엔드 서버(Node.js + WebSocket)와 기본 프론트엔드(Next.js + Tailwind)를 구축한다.

## Context
- Background: Asgard는 Odin(Claude), Brokkr(Codex), Heimdall(Gemini) 멀티에이전트 시스템. 현재 에이전트 실행 시 백그라운드에서 진행되어 사용자가 진행 상황을 확인할 수 없음. delegate-codex.sh와 delegate-gemini.sh가 artifacts/logs/에 로그를 기록하고, INDEX.md가 작업 상태를 추적함.
- Constraints: 외부 DB 없이 파일 시스템만 사용. localhost에서만 동작. 추가 인프라 없음.
- Dependencies: 없음 (첫 번째 Phase)
- Relevant files:
  - `artifacts/INDEX.md` — 작업 상태 SSoT
  - `artifacts/logs/execution.log` — 실행 로그
  - `scripts/delegate-codex.sh` — Brokkr 래퍼 (로그 출력 구조 참고)
  - `scripts/delegate-gemini.sh` — Heimdall 래퍼 (로그 출력 구조 참고)
  - `AGENTS.md` — 에이전트 규칙서 (상태 흐름 참고)

## Scope
### In
- `src/yggdrasil/` 디렉토리에 전체 코드 구현
- Node.js 백엔드 서버 (`src/yggdrasil/server/`)
  - Express + WebSocket(ws) 서버
  - `artifacts/logs/*.log` 파일 실시간 감시 (chokidar)
  - `artifacts/INDEX.md` 파싱 → 작업 상태 API
  - `artifacts/handoff/TP-*.md`, `RP-*.md` 내용 조회 API
  - WebSocket으로 로그 변경사항 실시간 푸시
  - 에이전트 프로세스 상태 감지 (PID 파일 기반)
- Next.js 프론트엔드 (`src/yggdrasil/dashboard/`)
  - Agent Status Cards — 3개 에이전트 상태(idle/running/blocked), 현재 TP, 모드, 경과 시간
  - Chronicle View — INDEX.md 기반 전체 작업 테이블 (상태별 아이콘/색상)
  - Live Log Panel — WebSocket으로 실시간 로그 스트리밍, 에이전트별 탭 전환, 자동 스크롤
  - TP/RP Viewer — 마크다운 렌더링 (클릭 시 사이드 패널)
  - 다크 모드 기본, Asgard 테마 색상
- `asgard dashboard` CLI 명령 (서버+프론트 동시 시작, 브라우저 자동 오픈)

### Out
- 에이전트 제어 기능 (Kill, Retry, Mode 전환) — TP-003에서 구현
- Timeline 뷰 — TP-003에서 구현
- 알림 시스템 (토스트, 사운드) — TP-003에서 구현
- 사용자 인증/원격 접속
- Docker/클라우드 배포

## Acceptance Criteria
- [ ] `asgard dashboard` 명령 실행 시 localhost:7777에서 대시보드 접속 가능
- [ ] Agent Status Cards에 Odin/Brokkr/Heimdall 3개 카드가 표시되고, 각각 idle/running/blocked 상태를 올바르게 표시
- [ ] Chronicle View에 INDEX.md의 Active Tasks와 Completed Tasks가 테이블로 표시됨
- [ ] delegate-codex.sh 또는 delegate-gemini.sh 실행 중일 때 해당 에이전트 카드가 running 상태로 전환되고 경과 시간이 표시됨
- [ ] Live Log Panel에서 실행 중인 에이전트의 로그가 실시간으로 스트리밍됨 (1초 이내 지연)
- [ ] 에이전트별 로그 탭 전환이 가능하고, 자동 스크롤 토글이 작동함
- [ ] TP/RP 클릭 시 마크다운이 렌더링된 사이드 패널이 열림
- [ ] 다크 모드 UI이고 Asgard 테마(북유럽 신화 색감)가 적용됨
- [ ] 서버가 artifacts/ 디렉토리 외부의 파일에 접근하지 않음 (보안)
- [ ] `npm run dev` (개발 모드)와 `npm run build && npm start` (프로덕션) 모두 작동

## Implementation Notes

### Architecture
```
src/yggdrasil/
├── server/                    # Node.js 백엔드
│   ├── index.ts               # Express + WebSocket 서버 진입점
│   ├── watcher.ts             # chokidar 파일 감시 (logs/, INDEX.md, handoff/)
│   ├── parser.ts              # INDEX.md 파싱, TP/RP 마크다운 파싱
│   ├── agents.ts              # 에이전트 상태 관리 (PID 감지, 상태 머신)
│   └── routes.ts              # REST API 라우트
├── dashboard/                 # Next.js 프론트엔드
│   ├── app/
│   │   ├── layout.tsx         # 루트 레이아웃 (다크모드, 폰트)
│   │   ├── page.tsx           # 메인 대시보드 페이지
│   │   └── globals.css        # Tailwind + 커스텀 스타일
│   ├── components/
│   │   ├── AgentCard.tsx      # 에이전트 상태 카드
│   │   ├── Chronicle.tsx      # INDEX.md 작업 테이블
│   │   ├── LiveLog.tsx        # 실시간 로그 패널
│   │   ├── DocViewer.tsx      # TP/RP 마크다운 뷰어
│   │   └── Header.tsx         # Yggdrasil 헤더
│   └── lib/
│       ├── websocket.ts       # WebSocket 클라이언트 훅
│       └── types.ts           # 타입 정의
├── package.json
└── tsconfig.json
```

### Backend API
```
GET  /api/status              → 전체 에이전트 상태 + INDEX 요약
GET  /api/chronicle           → INDEX.md 파싱 결과 (tasks 배열)
GET  /api/document/:type/:id  → TP/RP 마크다운 내용 (:type = tp|rp)
WS   /ws/logs                 → 실시간 로그 스트리밍 (agent 필터 지원)
WS   /ws/status               → 에이전트 상태 변경 실시간 푸시
```

### Agent State Detection
```
idle:     PID 파일 없음 + INDEX에 in-progress 없음
running:  PID 파일 존재 + 프로세스 살아있음
blocked:  INDEX에 blocked 상태 존재
done:     PID 파일 없음 + INDEX에 done/review-needed
```

PID 파일 위치: `artifacts/logs/.brokkr.pid`, `artifacts/logs/.heimdall.pid`
→ delegate-codex.sh, delegate-gemini.sh에서 실행 시 PID 기록, 종료 시 삭제

### Frontend Design
- 색상 팔레트: 다크 배경(#0f172a), 액센트(#3b82f6 Odin파랑, #f59e0b Brokkr주황, #10b981 Heimdall녹색)
- 폰트: JetBrains Mono (로그), Inter (UI)
- 레이아웃: 상단 Agent Cards → 좌측 Chronicle / 우측 Live Log → TP/RP 사이드 패널

### CLI Integration
install.sh의 `asgard` CLI에 `dashboard` 서브커맨드 추가:
```bash
cmd_dashboard() {
    cd "$ASGARD_PROJECT_DIR/src/yggdrasil"
    npm run dev &
    sleep 2
    open "http://localhost:7777" 2>/dev/null || xdg-open "http://localhost:7777" 2>/dev/null
}
```

### PID 파일 연동
delegate-codex.sh, delegate-gemini.sh에 PID 기록 로직 추가 필요:
```bash
# 실행 시
echo $CODEX_PID > artifacts/logs/.brokkr.pid
# 종료 시 (cleanup)
rm -f artifacts/logs/.brokkr.pid
```

## Risks / Edge Cases
- Next.js와 Express 서버를 하나의 프로세스로 통합할지, 별도 프로세스로 실행할지 → 개발 편의를 위해 커스텀 서버(Express에 Next.js 마운트) 방식 권장
- 로그 파일이 매우 클 경우 WebSocket 부하 → 마지막 500줄만 초기 전송, 이후 tail 방식
- INDEX.md 파싱 중 형식이 깨진 경우 → graceful 에러 처리, 마지막 유효 상태 유지
- macOS와 Linux에서 파일 감시 동작 차이 → chokidar이 플랫폼 차이를 추상화
- delegate 스크립트가 아직 PID를 기록하지 않으므로, 초기에는 프로세스명 기반 감지로 fallback
