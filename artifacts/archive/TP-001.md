# TP-001: Olympus 공개 배포 구조 구축

## Agent Target
codex

## Complexity Hint
moderate

## Objective
Olympus를 GitHub에서 one-line install로 배포 가능하도록 배포 구조(README, 원격 지원 install.sh, .gitignore, LICENSE, VERSION)를 구축한다.

## Context
- Background: 현재 Olympus는 로컬 디렉토리에서만 동작. `install.sh`는 소스 디렉토리 경로에 의존하므로 원격 실행(curl | bash) 시 실패함.
- Constraints: 외부 서비스 연동 없이 순수 GitHub 배포만 구현. npm publish 불필요.
- Dependencies: 없음
- Relevant files:
  - `install.sh` (현재 로컬 경로 의존 → 원격 지원으로 수정)
  - `CLAUDE.md`, `AGENTS.md`, `.claude/`, `scripts/` (README에 설명할 구조)

## Acceptance Criteria
- [ ] `curl -fsSL https://raw.githubusercontent.com/USER/olympus/main/install.sh | bash` 형식으로 실행 가능하도록 install.sh가 원격 환경을 감지하여 GitHub에서 파일을 다운로드함
- [ ] `bash install.sh` (로컬 실행)도 기존과 동일하게 작동함
- [ ] `README.md`에 설치 방법, 사용 방법, Skills 목록, 라이선스가 포함됨
- [ ] `README.md`에 `olympus new`, `olympus doctor`, `olympus update` 명령 설명 포함
- [ ] `LICENSE` 파일에 MIT 라이선스 전문 포함
- [ ] `.gitignore`에 `artifacts/handoff/`, `artifacts/logs/`, `shared/context.md`, `.DS_Store`, `src/` 포함
- [ ] `VERSION` 파일 내용: `0.1.0`

## Implementation Notes
- Approach:
  원격 실행 감지는 `BASH_SOURCE[0]` 또는 stdin 여부로 판단:
  ```bash
  if [ -p /dev/stdin ] || [ "${BASH_SOURCE[0]}" = "bash" ]; then
      # 원격 실행 — GitHub에서 파일 다운로드
      REMOTE=true
  else
      # 로컬 실행 — 현재 디렉토리의 파일 사용
      REMOTE=false
  fi
  ```
  원격 실행 시 `curl`로 GitHub raw URL에서 각 파일을 `/tmp/olympus-install/`에 다운로드 후 설치.
  BASE_URL은 `https://raw.githubusercontent.com/GITHUB_USER/olympus/main` (placeholder).

- File hints:
  - `README.md` → 프로젝트 루트
  - `LICENSE` → 프로젝트 루트
  - `.gitignore` → 프로젝트 루트
  - `VERSION` → 프로젝트 루트
  - `install.sh` → 기존 파일 수정 (기존 기능 유지)

```bash
# 원격 시뮬레이션 테스트

```

## Risks / Edge Cases
- 원격 실행 감지 로직이 모든 환경에서 신뢰할 수 없을 수 있음 → `--remote` 플래그 대안도 구현
- GitHub 저장소명/유저명이 아직 미정 → `GITHUB_USER`, `GITHUB_REPO` placeholder 사용, TODO 주석으로 명시
- `curl | bash` 보안 경고 → README에 대안(다운로드 후 검토) 방법 안내
